# Upgrade to 0.6.6 — register the memory-hygiene tick

You just upgraded mimir from `{from}` to `{to}`. 0.6.5 shipped the
`memory-hygiene` weekly scan and seeded `prompts/memory-hygiene.md`, but the
scheduled *tick* did not always reach existing homes (the defaults upgrade
didn't reconcile `scheduler.yaml` yet). 0.6.6 fixes that going forward and adds
`prompt_file` support to `add_schedule`. Make sure the tick is registered now.

This turn authorizes the schedule change below — adding this specific default
tick does not need separate operator escalation.

1. **Check whether the tick already exists.** Call `list_schedules` and look for
   a job named `memory-hygiene`. If it's present, there's nothing to do — skip to
   step 4 and end (idempotent; don't add a duplicate).

2. **If it's missing, add it with `add_schedule`.** Use the new `prompt_file`
   parameter so the tick matches the canonical bundled shape:

   ```
   add_schedule(
     name="memory-hygiene",
     cron="0 8 * * 2",          # Tuesdays 08:00 UTC
     prompt_file="memory-hygiene.md",
     priority="normal",
     channel_id=None,            # synthetic scheduler:memory-hygiene channel
   )
   ```

   `add_schedule` writes `scheduler.yaml` through the scheduler's own
   reload+validate path, so the tick is live immediately and durable across
   restart. Do **not** edit `scheduler.yaml` with `shell_exec` or the file tools
   to route around the path-confinement guard — that lands on disk but not in the
   running scheduler. If `add_schedule` reports the prompt file is missing, that
   means `prompts/memory-hygiene.md` wasn't seeded; surface that to the operator
   rather than inlining a prompt.

3. **Verify.** Call `list_schedules` again and confirm `memory-hygiene` now
   appears with `prompt_file: memory-hygiene.md`, the Tuesday cron, and `normal`
   priority.

4. **Report once.** Send a one-line result to the operator alert channel (pass an
   explicit `channel_id` — this is a non-interactive turn): whether the tick was
   already present, added now, or blocked (missing prompt file). If everything was
   already in place and there's nothing operator-actionable, end the turn silently.

This is a one-time migration nudge. Future default ticks will be proposed
automatically through the `scheduler.yaml` defaults-upgrade reconciliation.

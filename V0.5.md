# Mimir v0.5 вҖ” workspace monorepo + bundled MSAM

**Goal:** end the cross-repo coordination tax between mimir and msam2 by merging the MSAM source tree into mimir as a workspace package, then close the operator-UX gap so `mimir setup && mimir run` brings up a working agent + memory pair without separate MSAM install ceremony.

**Frame:** v0.4 made mimir self-aware about resources (cost, plan windows, subagent spend). The next blocker on velocity is structural вҖ” every PR that touches the mimir/MSAM boundary today is a two-repo coordination dance, and every new operator who installs mimir hits a wall when MSAM isn't running. v0.5 makes both go away.

## Pieces (high-leverage to small)

1. [Workspace monorepo merge](#1-workspace-monorepo-merge) вҖ” filter-repo msam2 into mimir at `/msam/`, set up uv workspace, archive msam2. The structural piece.
2. [Bundled MSAM lifecycle](#2-bundled-msam-lifecycle) вҖ” `mimir setup` generates a shared API key into both configs, `mimir run` supervises MSAM as a child process, opt-out via `MIMIR_SPAWN_MSAM=false` for external-MSAM deployments.
3. [Integration bench harness](#3-integration-bench-harness) вҖ” end-to-end benches that exercise MSAM through mimir's actual call sites, complementing MSAM-only synthetic benches.
4. [muninnbot pin + transition note](#4-muninnbot-pin) вҖ” pin muninnbot's docker to a known msam2 source commit before archiving, so its build keeps working through the deprecation window.
5. [Doc cleanup](#5-doc-cleanup) вҖ” V0.4.md path refs, SPEC.md cross-refs, README, FUTURE_WORK В§11 update.

**Deferred to v0.6+:**
- Per-channel persona вҖ” design from v0.4's deferred list, ~50 LOC change in `load_core` to take channel_id. Independent of the merge work; pick up when feature priorities call for it.
- Temporal index for retrieval (FUTURE_WORK В§3.1) вҖ” dominant probe-type miss in bench data; needs MSAM-side schema work that's much easier post-merge.
- Reply / repost graph (FUTURE_WORK В§3.3) вҖ” weakest probe types; similar dependency pattern.
- Belief-revision / claims-audit reflection track (FUTURE_WORK В§8.3 partial) вҖ” extends the v0.4 reflection skill once the operator and agent have settled patterns from a few weeks of running it.

### History preservation

`git filter-repo` extracts msam2's `msam/` subdir history into a temporary repo, then merge as a new tree:

```bash
# 1. Clone msam2 fresh (don't use the active worktree).
git clone https://github.com/jasoncarreira/msam /tmp/msam-export
cd /tmp/msam-export

# 2. Filter to just the msam/ source dir + benchmarks/ + tests/ (drop bench
#    data, results, external symlinks вҖ” the 15GB tail).
git filter-repo \
    --path msam/ \
    --path benchmarks/ \
    --path tests/ \
    --path pyproject.toml \
    --path README.md \
    --path msam.example.toml \
    --to-subdirectory-filter msam

# 3. In mimir, merge as a separate root.
cd ~/projects/odin/mimir
git remote add msam-import /tmp/msam-export
git fetch msam-import
git merge --allow-unrelated-histories msam-import/main \
    -m "Import MSAM source under msam/ via filter-repo

History preserved from github.com/jasoncarreira/msam main branch.
Bench data + results dropped at filter time (stays in archived msam2)."

# 4. Validate: msam/ has full file history, no large files imported.
git log --oneline --follow msam/msam/server.py | head
du -sh msam/

# 5. Bring over the hindsight-ideas branch.
git fetch msam-import refs/heads/hindsight-ideas:msam-hindsight-ideas
# Now msam-hindsight-ideas exists in mimir as a regular branch.
```

### Workspace config

Root `pyproject.toml`:

```toml
[tool.uv.workspace]
members = ["mimir", "msam"]

[tool.uv.sources]
mimir = { workspace = true }
msam = { workspace = true }
```

Each package keeps its own `pyproject.toml` with its own deps, version, etc. `uv sync` at root installs both editable; either can still publish to PyPI independently.

**Versioning:** single version line вҖ” mimir's git tag IS msam's version. Both packages' `pyproject.toml` versions track the tag. Trade-off: msam-only changes don't get distinct version numbers, but msam2 had no real PyPI consumers, so this is theoretical cost only.

3. msam-hindsight-ideas worktree at `/Users/jcarreira/projects/odin/msam-hindsight-ideas/` cleaned up вҖ” its contents are now in mimir's `msam-hindsight-ideas` branch.

### Tests

## 2. Bundled MSAM lifecycle

**Goal:** new operator runs `mimir setup --home X && mimir run --home X` and gets a working agent + memory pair, no separate MSAM install ceremony. External MSAM deployments still work via `MSAM_ENDPOINT` override.

### Setup-time integration

`mimir setup` post-merge gains MSAM-aware behavior:

1. Detects whether `msam` is importable in the current Python env (it always will be in the workspace install).
2. Generates a single API key (`secrets.token_urlsafe(32)`).
3. Writes the key to mimir's `<home>/.env` as `MSAM_API_KEY=...`.
4. Writes a generated `<home>/msam.toml` with:
   - The same API key under `[server].api_key`
   - `[server].port = <auto-pick free port>` (default 3002, fall back to next free)
   - `[storage].db_path = <home>/.mimir/msam.db`
   - Sensible embedding-model + retrieval defaults (copied from `msam.example.toml`)
5. Writes mimir's `MSAM_ENDPOINT=http://localhost:<chosen port>` matching.

Existing keys preserved on re-run, like the existing MIMIR_API_KEY behavior.

### Run-time supervisor

`mimir run` post-merge gains an MSAM child-process supervisor when `MIMIR_SPAWN_MSAM=true` (default `true` if `msam` is importable; default `false` if not).

```python
# mimir/msam_supervisor.py вҖ” new module
class MsamSupervisor:
    """Forks msam.server, monitors, restarts on crash with backoff,
    pipes stderr into mimir's logger, graceful-shuts on parent signal."""

    async def start(self) -> None:
        # spawn, wait for /v1/health to return 200
        ...

    async def _monitor(self) -> None:
        # poll process; on exit, restart with exponential backoff
        # cap at N restarts in M minutes; past that, log critically
        # and stop trying (operator's problem)
        ...

    async def stop(self) -> None:
        # SIGTERM, wait for graceful shutdown, SIGKILL if needed
        ...
```

Supervisor lifecycle hooks into mimir's existing aiohttp app startup/shutdown handlers in `server.py`.

### Health check + visibility

- Startup log: `"MSAM supervisor: spawned PID 12345 on port 3002, /v1/health OK"` or `"MSAM_ENDPOINT=external; not spawning"`.
- New event types: `msam_supervisor_spawned`, `msam_supervisor_restarted`, `msam_supervisor_exited`. Surfaced in algedonic block via `feedback.py` rules so the agent sees if MSAM is unhealthy.
- `mimir stats` gains an MSAM line: process status, uptime, restart count.

### Opt-out paths

- `MIMIR_SPAWN_MSAM=false` вҖ” disable spawn entirely.
- `MSAM_ENDPOINT` set to non-localhost вҶ’ automatically disable spawn (operator pointing at external MSAM).
- `mimir setup --no-msam` flag вҶ’ setup skips MSAM config generation.

### Tests

- `tests/test_msam_supervisor.py`:
  - Spawn happy path (mock subprocess)
  - Health-check timeout вҶ’ supervisor surfaces error
  - Crash + restart with backoff
  - Graceful shutdown on parent stop
  - External MSAM endpoint вҶ’ supervisor doesn't spawn

### Effort

- `MsamSupervisor` module: ~150 LOC
- Setup integration: ~80 LOC (key generation, msam.toml writer)
- Tests: ~150 LOC (subprocess mocking is the bulk)
- `feedback.py` rule additions: ~10 LOC

**Total: ~390 LOC. ~2 focused days.**

## 3. Integration bench harness

**Goal:** end-to-end benches that catch MSAM-mimir interactions invisible to MSAM-only synthetic benches. Enables future retrieval-quality work to verify wins are still wins through mimir's actual pipeline.

### What's invisible to MSAM-only benches today

MSAM's existing benches measure synthetic-query вҶ’ MSAM вҶ’ atom-list quality. They don't see:

- **Cache interaction.** Mimir's prompt cache changes which atoms get re-queried turn-to-turn; MSAM-only benches don't model cache behavior.
- **Contextual rewrite.** When mimir passes `context=` (last 10 messages), MSAM's `enable_contextual_rewrite` may rewrite "yes, look for that" into self-contained form. The bench should measure end-to-end retrieval quality with rewrite on/off.
- **Mark-contributions credit pass.** After a turn, mimir credits atoms that contributed to the response. The credit signal feeds future retrieval ranking. Bench should measure whether the credit signal is improving retrieval over time вҖ” visible only end-to-end.
- **Session boundaries.** mimir's session lifecycle drives which `session_id` gets passed to MSAM; bench should measure retrieval quality across session boundaries (within-session vs. cross-session recall).

### Harness shape

```
msam/benchmarks/
в”ңв”Җв”Җ synthetic/                  (MSAM-only, existing)
в””в”Җв”Җ integration/                (NEW вҖ” mimir + MSAM end-to-end)
    в”ңв”Җв”Җ runner.py               # spawns mimir + MSAM in test-mode
    в”ңв”Җв”Җ corpus_seed.py          # builds bench-specific channel histories
    в”ңв”Җв”Җ probes.jsonl            # bench probes (query, expected atoms)
    в””в”Җв”Җ score.py                # scores retrieval quality through mimir
```

The runner uses mimir's existing `BenchBridge` (the in-process bridge designed for this) so bench probes hit the same dispatch path as live messages.

### What this unlocks for v0.6+

- **Temporal-index work** (FUTURE_WORK В§3.1) becomes measurable end-to-end. Bench shows whether the new index improves "what did Alice say last week" probes through mimir's prompt assembly, not just through raw MSAM queries.
- **Reranker work** (FUTURE_WORK В§7.5) gets the same treatment.
- **Cache hit rate** correlations: does adding metadata X to atoms hurt cache hit rate? Bench measures.

### Effort

- Harness scaffolding: ~200 LOC
- A first probe set adapted from existing MSAM bench probes: ~50 LOC + JSONL data
- Integration with existing CI: ~30 LOC
- Tests for the harness itself: ~80 LOC

**Total: ~360 LOC + probe data. ~1.5 focused days.**

## Bench predictions

v0.5 is structural вҖ” no direct bench-quality impact expected. The win shows up indirectly:

- В§3 integration bench harness exists and is exercised. Future v0.6+ retrieval changes get measured end-to-end.
- В§2 bundled MSAM removes a class of "MSAM is down" failures that today degrade silently into empty `recent_session_boundaries` / `most_retrieved_atoms` returns.
- Operationally: time-to-first-working-agent for a new operator drops from "install MSAM, configure msam.toml, start MSAM, sync API key, edit mimir's .env, start mimir" to "install mimir, run setup, run run."
